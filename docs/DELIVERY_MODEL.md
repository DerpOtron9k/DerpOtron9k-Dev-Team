# Delivery Model

## Purpose

This repository starts new projects with a lightweight Scrum-shaped AI delivery model.

## Authority

- The user is the Product Owner unless they explicitly choose otherwise.
- The starter delivery ring helps structure kickoff and early backlog flow.
- The project-specific architecture council is selected during kickoff.

## Starter Artifact Hierarchy

1. `PRD`
   - File: [.taskmaster/docs/prd.txt](/R:/GitRepo/DerpOtron9k-Dev-Team/.taskmaster/docs/prd.txt)
2. `Epics`
   - File: [docs/EPICS.md](/R:/GitRepo/DerpOtron9k-Dev-Team/docs/EPICS.md)
3. `PBIs / User Stories`
   - tracked in Task Master after kickoff
4. `Implementation Tasks`
   - subtasks under active PBIs
5. `Sprint`
   - File: [docs/SPRINT.md](/R:/GitRepo/DerpOtron9k-Dev-Team/docs/SPRINT.md)

## Default Rule

Use the root delivery ring to get started, then shape the real project council and backlog based on the project's actual needs.

## Default Tooling Assumption

- Serena and Context7 are treated as default MCP staples for code-aware work and current documentation retrieval.
