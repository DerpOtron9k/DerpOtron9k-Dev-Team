# Ceremonies

## Kickoff

The root repo is designed to start with:

```text
Start Dev Team Kickoff
```

Kickoff should:

- confirm the Product Owner
- request and confirm explicit Product Owner direction before framing the product
- stop in a waiting state if direction is still unclear
- define non-goals
- confirm Serena and Context7 availability or identify the setup gap
- select the project council
- establish a first PRD, epic map, and sprint recommendation

Kickoff must not infer direction from repo name, example overlays, reference blueprints, or other non-authoritative repo signals.

## Turn-level Scrum

Run every meaningful implementation turn:

- what changed
- what is blocked
- next highest-value slice

## Backlog Refinement

Run when:

- next PBIs are unclear
- priorities changed
- a completed slice changes what should come next

## Sprint Planning

Run when:

- the sprint goal is complete
- the sprint no longer makes sense
- in-sprint PBIs are not agreed
