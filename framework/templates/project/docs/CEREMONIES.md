# Ceremonies

## Turn-level Scrum

Run every meaningful implementation turn.

Output:

- what changed
- what is blocked
- consensus or disagreement when relevant
- next highest-value slice

Before early delivery work, confirm whether Serena and Context7 are available if the project expects this default MCP stack.

## When To Run Backlog Refinement

Run backlog refinement when any of these are true:

- 2 to 4 PBIs have been completed since the last refinement
- the Product Owner changes priority or product direction
- a top-level Task Master item is clearly too large, vague, or implementation-shaped
- the next 2 to 3 PBIs are not ready for execution
- a completed slice changes what should logically come next

## When To Run Sprint Planning

Run sprint planning when any of these are true:

- the current sprint goal is complete
- the current sprint no longer makes sense because priority changed
- the current sprint has too much unfinished work and needs an explicit reset
- the next in-sprint PBIs are not yet agreed

## Review

Run at sprint boundary or when a sprint goal is effectively met.

## Retrospective

Run after review, but keep it short.

## Practical Rule

If there is no meaningful checkpoint, do not stop to hold a ceremony.
